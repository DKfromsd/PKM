

토큰 수가 기하급수적으로 늘어나는 **Long-Context 추론** 시대에는 단순히 "데이터를 저장하는 것"을 넘어, 메모리 계층 구조(Memory Hierarchy) 전체의 데이터 흐름(Data Orchestration)이 커널 레벨에서 어떻게 제어되느냐가 핵심.

---
### **1. 커널 레벨에서의 3대 핵심 역할 분담**

토큰이 늘어날수록 KV Cache(Key-Value Cache)의 크기가 비대해지며, 이를 처리하기 위해 CPU, SSD, NAND는 다음과 같은 커널 기반의 역할을 수행해야 합니다.

#### **① CPU: 데이터 오케스트레이터 및 가상 메모리 관리자**

- **역할:** 과거처럼 단순 연산만 하는 게 아니라, **커널 레벨의 페이징(Paging) 및 스와핑(Swapping) 엔진** 역할을 합니다.
    
- **커널 동작:**
    
    - **KV Cache 관리:** HBM(G1) -> DRAM(G2) -> SSD(G3)로 이어지는 계층형 저장 구조에서 데이터의 '우선순위'를 결정합니다.
        
    - **Direct Storage Access:** CPU 오버헤드를 줄이기 위해 커널의 파일 시스템을 거치지 않고 장치 간에 직접 데이터를 전송하는 NVMe-oF(NVMe over Fabrics)나 **GPUDirect Storage** 같은 기술을 커널 레벨에서 제어합니다.
        
    - **추론 에이전트 지시:** CPU는 에이전트의 지시를 받아 '어느 토큰 뭉치가 지금 연산에 필요한가?'를 판단하고, 필요한 데이터를 미리 SSD에서 DRAM으로 끌어올리는 프리페칭(Prefetching)을 수행합니다.
        

#### **② SSD: 영구 저장소를 넘어선 '확장 메모리(Extended Memory)'**

- **역할:** 단순한 저장장치가 아니라 **커널이 인식하는 거대한 스왑 공간(Swap Space)** 또는 **휘발성 메모리의 연장선**으로 동작합니다.
    
- **커널 동작:**
    
    - **추론 컨텍스트 저장(ICMS):** 엔비디아의 Vera Rubin 플랫폼에서 보여준 것처럼, **ICMS(Inference Context Memory Storage)** 레이어를 통해 수조 개의 토큰 컨텍스트를 SSD에 상주시키고 필요할 때 커널 인터럽트를 최소화하며 고속으로 로딩합니다.
        
    - **순차적 읽기 최적화:** AI 추론은 무작위 읽기(Random IO)보다 거대한 매트릭스를 읽어오는 순차적 읽기(Sequential Read)가 중요합니다. 커널은 SSD 컨트롤러와 협력하여 데이터 파이프라인을 일직선으로 정렬합니다.
        

#### **③ NAND: 하이 대역폭 플래시(HBF)로의 진화**

- **역할:** 기존의 느린 저장 매체에서 HBM의 용량 한계를 극복하는 '대용량 대역폭 매체'로 변모합니다.
    
- **커널 동작:**
    
    - **병렬 데이터 경로:** 샌디스크(SNDK)가 추진하는 **HBF(High Bandwidth Flash)** 기술은 NAND 내부에 수많은 병렬 경로를 만들어, 커널 레벨에서 DRAM 수준의 대역폭을 흉내 냅니다.
        
    - **커널 우회(Bypass):** 커널의 무거운 스택을 거치지 않고 하드웨어 레벨에서 직접 데이터를 전송하는 기술이 NAND 컨트롤러 수준에서 구현되어야 합니다.

| **요소**     | **기존 방식 (Training/General)** | **향후 추론 방식 (Inference/Long-Context)** |
| ---------- | ---------------------------- | ------------------------------------- |
| **I/O 패턴** | 무작위 작은 파일 (IOPS 중시)          | **거대한 시퀀스 데이터 (Throughput 중시)**       |
| **데이터 이동** | SSD → CPU → DRAM → GPU       | **SSD → GPU (CPU/커널 우회)**             |
| **메모리 인식** | 저장장치와 메모리를 분리                | **SSD를 '레이턴시가 긴 RAM'으로 인식**           |
| **핵심 지표**  | Read/Write 속도                | **Tokens per Second (TPS) 최적화**       |
