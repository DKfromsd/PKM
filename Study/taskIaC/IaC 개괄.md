## 1. 분야별 API 종류 및 사용법

- **웹 (Web):** **REST API**나 **GraphQL**을 사용. HTTP 통해 데이터 교환, 브라우저와 서버 간의 통신에 필수.
- **임베디드 (Embedded):**  **HAL API**나 **C 표준 라이브러리** 사용. 센서,모터 제어.
- **빅데이터:** **REST API** 외에도 대량데이터 스트리밍 **gRPC**나 프레임워크(Spark, Kafka) 전용 API를 사용
- **사물인터넷 (IoT):** 전력 소모 적은 **MQTT**나 **CoAP** 기반 API를 사용해 기기 간 상태 공유.
- **물리 인공지능 (Robotics/AI):** **ROS API**나 **TensorFlow/PyTorch API**를 통해 실시간 데이터 처리. 

## 2. DevOps 핵심 도구의 이유

- **Go:** 이 모든 도구(Docker, K8s, Terraform)의 모태. 컴파일 속도, 동시성 처리 인프라 도구 제작에 최적.
- **Docker:** 앱환경에 구애받지 않게 컨테이너화하여 개별환경 문제 해결.
- **Terraform:** 코드로 인프라를 관리(IaC). 클릭 대신 스크립트로 서버 수백 대 동시 세팅.
- **Prometheus:** 시스템(CPU, 메모리, 트래픽)를 시계열 데이터로 수집하고 감시하는 '눈' 역할 

## 3. 언어별 API 활용 예시 및 융합

이 기술들은 **'Go로 만든 도구(Docker/Terraform) 위에 각 언어로 짠 앱을 올리고, Prometheus로 감시하는'** 방식으로 융합됩니다. 

| 언어               | 주요 역할       | API 예시                                             |
| ---------------- | ----------- | -------------------------------------------------- |
| **TypeScript**   | 웹 프론트엔드/백엔드 | **Fetch API**를 사용하여 서버에서 데이터를 가져와 화면에 렌더링.         |
| **Go**           | 고성능 마이크로서비스 | **gRPC API**를 구현하여 서비스 간에 초고속으로 데이터를 교환.           |
| **Python**       | AI 모델 배포    | **FastAPI**를 통해 딥러닝 모델의 추론 결과를 웹 API 형태로 제공.       |
| **Java**         | 대규모 기업용 시스템 | **Spring Boot**의 REST API를 사용하여 안정적인 데이터 처리 서버 구축. |
| **JS (Node.js)** | 실시간 통신      | **Socket.io API**를 사용하여 IoT 기기와의 실시간 양방향 통신 구현.    |

**융합 시나리오:**

1. **Terraform**으로 클라우드 서버를 할당.
2. **Go**로 짠 백엔드와 **Python** AI 모듈을 **Docker**로 .
3. 서비스 간 데이터는 **REST/gRPC API**로 소통.
4. 이 과정에서 발생하는 모든 지표는 **Prometheus**가 수집 대시보드
